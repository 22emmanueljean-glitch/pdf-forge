System Hierarchy

Control chain: Inputs → Metrics → Triggers → Automations → Operator Oversight → Investor Reporting. Each layer produces verifiable signals and audits the layer beneath it.

  • Signals: raw → normalized → composite (LPI)
  • Controls: thresholds → triggers → actions
  • Assurance: logs → receipts → monthly brief

Decision Protocols

Protocol path: Metric Breach (auto-detected) → Automated Response (playbook action) → Human Oversight (confirm/override) → Investor Report (weekly capital brief). Timing windows lock to 48 h end-to-end.

  • Escalation: if 2× repeat within 72 h → incident review
  • Freeze rules: ad expansion locked if reserves < 10%
  • Rebalance: capital shifts follow MER/CAC ladders

Data Provenance Chain

Layer 6 Playbooks — Profit Restoration Protocols

6.1 CAC Spike

Rank by ROAS → pause bottom 20% → duplicate top creative to new audience → recheck in 48h.

6.2 Churn Surge

Export cancels → tag cohort → retention offer + support callback → measure 7d.

6.3 Automation Failure

Identify failing job → retry → fallback path → rotate token → verify heartbeat in 15m.

6.4 Refund Spike

Read reason codes → route to shipping/product/support → implement fix → recheck next cycle.

6.5 Lead Drop

CTR vs impressions → swap creative (fatigue) or widen audience (burnout).

6.6 Margin Compression

Profit ↓, revenue flat → audit SaaS, renegotiate vendors, refresh creatives.

6.7 Scale Surge

Revenue +25% → fulfillment audit + lock 10% to reserves → guard CX.

6.8 Liquidity Lock

Reserves <10% → freeze ad expansion until reserve ≥15%.

Failure Modes & Recovery Paths

  • Network/API: backoff-retry (3/30/300s), token rotation on 401, fallback provider; incident only if 3 cycles fail.
  • Data drift: rolling baseline recalculation; freeze anomaly until two consecutive confirmations.
  • Deadlocks: watchdog restarts job; escalates if >2 restarts in 1h.
  • Human delay: secondary ping after 15m; auto-approve low-risk rebalance under $X threshold.
